# C26166
[!INCLUDE[vs2017banner](../code-quality/includes/vs2017banner.md)]

avertissement C26166 : Éventuellement acquérir ou non détenir le verrou \<lock\> dans la fonction \<func\>.  
  
 L'avertissement C26166 ressemble à l'acertissement [C26116](../code-quality/c26116.md), excepté que le niveau de confiance est plus bas.  Par exemple, la fonction peut contenir des erreurs d'annotation.  
  
## Exemple  
 Le code suivant affiche le code qui génère C26166 d'avertissement.  
  
```cpp  
  
typedef struct _DATA {   
CRITICAL_SECTION cs;   
} DATA;   
  
_Acquires_lock_(p->cs) void Enter(DATA* p) {   
    EnterCriticalSection(&p->cs); // OK   
}   
  
_Acquires_lock_(p->cs) void BAD(DATA* p) {} // Warning C26166  
  
```
