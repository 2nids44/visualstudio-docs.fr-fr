# <a name="c6516"></a>C6516
avertissement C6516 : annotation non valide : aucune propriété spécifiée pour \<nom > attribut  
  
> [!NOTE]
>  Cet avertissement se produit uniquement dans le code qui utilise une version obsolète de la langue d’annotation de code source (SAL). Nous vous recommandons de porter votre code pour utiliser la dernière version de SAL. Pour plus d’informations, consultez [à l’aide d’Annotations SAL pour réduire les défauts du Code C/C++](../code-quality/using-sal-annotations-to-reduce-c-cpp-code-defects.md).  
  
 Cet avertissement indique qu’aucune propriété n’a été spécifiée dans l’attribut ou la propriété qui a été spécifiée est non valide ; Par conséquent, l’attribut ne peut pas être considérée comme terminée.  
  
## <a name="example"></a>Exemple  
 Le code suivant génère cet avertissement car Deref = 1 spécifie uniquement le niveau d’indirection, mais ces informations uniquement ne permettent pas l’outil d’analyse :  
  
```  
// C  
#include <CodeAnalysis\SourceAnnotations.h>  
void f([SA_Pre(Deref=1)] char* pc);  
  
// C++  
#include <CodeAnalysis\SourceAnnotations.h>  
using namespace vc_attributes;  
void f([Pre(Deref=1)] char* pc);  
  
```  
  
 Pour corriger cet avertissement, une autre propriété, tels que l’accès est requis pour indiquer à l’outil d’analyse, ce qui doit être appliqué sur les éléments non référencés. Le code suivant corrige cet avertissement :  
  
```  
// C  
#include <CodeAnalysis\SourceAnnotations.h>  
void f([SA_Pre(Deref=1, Access=SA_Read)] char* pc);   
  
// C++  
#include <CodeAnalysis\SourceAnnotations.h>  
using namespace vc_attributes;  
void f([Pre(Deref=1, Access=Read)] char* pc);  
```
