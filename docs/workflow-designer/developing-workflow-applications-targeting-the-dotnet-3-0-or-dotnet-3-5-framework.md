# <a name="developing-workflow-applications-targeting-the-net-30-or-net-35-framework"></a>Développement d'applications de workflow qui ciblent le .NET 3.0 ou .NET 3.5 Framework
[!INCLUDE[vs2010](../misc/includes/vs2010_md.md)] fournit le Concepteur de flux de travail Windows pour la création de [!INCLUDE[wf](../workflow-designer/includes/wf_md.md)] les applications qui ciblent le [!INCLUDE[netfx40_long](../workflow-designer/includes/netfx40_long_md.md)]. Le concepteur utilise l'interface utilisateur [!INCLUDE[vsprvs](../code-quality/includes/vsprvs_md.md)] que les utilisateurs connaissent bien pour la construction graphique et le débogage d'applications de workflow.

 [!INCLUDE[vs2010](../misc/includes/vs2010_md.md)] fournit également un accès à un concepteur de workflow hérité qui peut être utilisé pour cibler le [!INCLUDE[netfx35_long](../workflow-designer/includes/netfx35_long_md.md)] ou le [!INCLUDE[vstecwinfx](../workflow-designer/includes/vstecwinfx_md.md)]. Cette section propose de l'aide relative à l'utilisation de ce concepteur hérité.
